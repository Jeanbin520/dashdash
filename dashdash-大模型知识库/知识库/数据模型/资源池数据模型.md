# 资源池数据模型

上级：[[dashdash-大模型知识库/地图/知识库地图|知识库地图]]

## 结论

资源池是Sub2API分组在本地系统中的统计实体。`resource_pools`以`source + external_group_id`唯一约束绑定外部来源分组；不同计量口径、订阅类型和资源池不跨单位汇总。

`percentage_pool`不保存`total_quota`。共享模型的百分比资源池计算只使用状态为`active`、数据新鲜且额度在0至100之间的账号；其他账号计入不可用账号数。

`accounts`保存当前账号快照，`resource_pool_memberships`保存分组归属的生效区间，`usage_records`在写入时固定关联当时的`resource_pool_id`。账号后续迁组不会改变历史用量归属。

## SQLite 方言映射

数据库已从 PostgreSQL 迁移至 SQLite（libSQL）。Schema 使用 `drizzle-orm/sqlite-core`，类型映射如下：

| PostgreSQL | SQLite | 说明 |
| --- | --- | --- |
| `uuid().defaultRandom()` | `text().$defaultFn(() => crypto.randomUUID())` | SQLite 无原生 UUID，用 text 存储 |
| `numeric(p, s)` | `numeric()` | SQLite 忽略精度，按 TEXT 存储 |
| `jsonb().$type<T>()` | `text({ mode: "json" }).$type<T>()` | JSON 存为 text，Drizzle 自动序列化 |
| `timestamp({ withTimezone: true })` | `integer({ mode: "timestamp" })` | 存 Unix 秒，读出为 `Date` 对象 |
| `boolean()` | `integer({ mode: "boolean" })` | SQLite 无原生布尔类型 |
| `varchar({ length })` | `text()` | SQLite 的 TEXT 不限长度，长度约束仅在应用层 |

## 重要性

该模型为同步、倍率结算、摘要接口和Dashboard提供同一口径。用量记录同时保存原始调用量、时段倍率、模型倍率和加权结果，规则更新不会隐式改写历史结算。

## 验证

- `pnpm.cmd --filter @dashdash/shared test`
- `pnpm.cmd check`
- `pnpm.cmd --filter @dashdash/api db:generate`

## 备注

- 低额度阈值必须由资源池规则或调用方显式传入，当前不固化产品默认值。
- Sub2API接口字段仍未验证；连接器必须将外部字段映射隔离在适配层。
- SQLite 的 `numeric` 列读出为 `string`，与共享模型 `ResourcePool` 中的 `number` 类型存在差异，映射层需做数值转换。
- `timestamp` 列读出为 `Date` 对象，与共享模型中的 `string` 类型存在差异，映射层需做 `.toISOString()` 转换。
