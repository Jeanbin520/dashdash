# 数据库接入模式

上级：[[dashdash-大模型知识库/地图/知识库地图|知识库地图]]

## 结论

数据库通过 Drizzle ORM + `@libsql/client` 接入 NestJS 依赖注入容器，采用三层结构：

1. **Schema**（`schema.ts`）：使用 `drizzle-orm/sqlite-core` 定义表结构，与驱动解耦。
2. **客户端工厂**（`db.ts`）：创建 libsql 客户端并包装为 Drizzle 实例，导出 `Database` 类型和 `DATABASE_TOKEN`（Symbol）注入令牌。
3. **NestJS 模块**（`database.module.ts`）：用工厂模式注册 DB 实例，导出令牌供其他模块注入。

核心代码模式：

```ts
// db.ts - 工厂 + 令牌
export type Database = LibSQLDatabase<typeof schema>;
export const DATABASE_TOKEN = Symbol("DATABASE");
export function createDatabase(): Database {
  const url = process.env.DATABASE_URL ?? "file:./dashdash.db";
  const client = createClient({ url });
  return drizzle(client, { schema });
}

// database.module.ts - NestJS 模块
@Module({
  providers: [{ provide: DATABASE_TOKEN, useFactory: () => createDatabase() }],
  exports: [DATABASE_TOKEN],
})

// controller/service - 注入使用
constructor(@Inject(DATABASE_TOKEN) private readonly db: Database) {}
```

## 重要性

这是后续所有 CRUD 端点、同步服务和数据查询的基础模式。新增任何需要数据库的模块时，注入 `DATABASE_TOKEN` 即可，无需重复创建连接。

## 验证

- `pnpm.cmd --filter @dashdash/api check` - 类型检查通过
- `pnpm.cmd --filter @dashdash/api build` - 构建通过
- `curl http://localhost:3000/health/db` 返回 `{"status":"ok","resourcePools":0}`

## 备注

- `DATABASE_URL` 必须使用 `file:` 前缀（如 `file:./dashdash.db`）。
- `Database` 类型是 `LibSQLDatabase<typeof schema>`，不是 `BaseSQLiteDatabase`。
- 项目使用 `module: NodeNext` + ESM，所有相对导入必须带 `.js` 后缀（如 `./schema.js`），包导入不带后缀。
- 当前未加 `onModuleDestroy` 关闭连接；libsql 本地文件模式无需显式关闭。若未来换远程 Turso 再补。
- 迁移通过 `pnpm.cmd --filter @dashdash/api db:migrate` 应用，迁移文件在 `apps/api/drizzle/` 目录。
